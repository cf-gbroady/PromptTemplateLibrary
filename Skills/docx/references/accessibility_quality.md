# DOCX Accessibility and Quality Checklist

Use this checklist for Word documents that will be shared, published, converted to PDF, or consumed with assistive technology.

## Structure

- Use Word heading styles instead of manually bolding or enlarging text.
- Do not skip heading levels without a reason.
- Use built-in list styles for lists.
- Use real tables, not tab-aligned text.
- Add captions or explanatory lead-ins for complex tables and figures.

## Images

- Add meaningful alt text for informative images.
- Mark decorative images as decorative when the authoring tool supports it.
- Avoid embedding text-heavy screenshots unless the text is also present in the document body.

## Tables

- Use a simple header row when possible.
- Avoid split or nested tables unless necessary.
- Avoid merged cells in data tables unless the user explicitly needs them.
- Verify table reading order manually for complex tables.

## Links

- Use descriptive link text, not bare URLs, unless the URL itself is the content.
- Verify external links are intended and safe to include.

## Metadata

- Set title and subject when useful.
- Avoid secrets, credentials, personal identifiers, or confidential internal notes in core properties.
- Use a clear language setting when supported by the authoring workflow.

## Quality Review

Before final delivery:
1. Re-open or inspect the generated file.
2. Confirm headings, tables, images, and links are present as expected.
3. Report any manual checks still needed in Microsoft Word, such as Accessibility Checker results or alt text verification.
4. If the DOCX will become a PDF, fix accessibility in the DOCX source before exporting whenever possible.
